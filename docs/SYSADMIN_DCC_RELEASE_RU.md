# Инструкция для системного администратора (DCC)

## Что включено в релиз

- Увеличено время жизни access-сессии до 60 минут.
- Исправлен workflow замечаний:
  - сценарий `I -> FINAL_CONFIRM -> только A`;
  - повторные клики по ключевым действиям (`AP`, `PUBLISH`, `FINAL_CONFIRM`) безопасны (idempotent).
- Для блока carry-over (`Должны были устранить`) решение `OPEN/CLOSED` фиксируется один раз и больше не меняется.
- Исправлено правило AP: если все замечания отклонены LR, AP разрешен.
- Обновлен расчет статуса по замечаниям (учитываются только релевантные замечания подрядчика).
- Обновлен UI на вкладках `Обзор` и `Проекты` по комментариям коллег.

## Быстрый запуск (локальный smoke)

1. Backend:
   - `cd backend`
   - `PYTHONPATH=. uvicorn app.main:app --host 0.0.0.0 --port 8017`
2. Frontend:
   - `cd frontend`
   - `VITE_API_URL=http://localhost:8017 npm run dev -- --host 0.0.0.0 --port 3010`
3. Вход:
   - URL: `http://localhost:3010`
   - Логин: `admin@ivamaris.io`
   - Пароль: `admin123`

## Минимальный чек-лист после выкладки

- Авторизация admin проходит, сессия не выбивает в течение 60 минут.
- В `Обзор`:
  - столбец называется `Дата создания`;
  - виден блок профиля (ФИО) и таблица ролей в проектах.
- В `Проекты -> Участники проекта`:
  - роли отображаются бизнес-названиями (`ТДО разработчика`, `R/LR заказчика`, ...);
  - у пользователей без прав скрыты лишние столбцы и кнопка добавления.
- В `Проекты -> Реестр документов (MDR)` отсутствует столбец `ID`.
- Workflow:
  - `AP` можно поставить, если замечания LR отклонены;
  - `I -> FINAL_CONFIRM -> второй I запрещен, A разрешен`;
  - carry-over: `OPEN/CLOSED` фиксируется с первого выбора и дальше не меняется.

## Диагностика

- Если в форме логина `Failed to fetch`:
  - проверить, что фронт смотрит в правильный `VITE_API_URL`;
  - проверить CORS (для localhost разрешен любой порт).
- Если действие не выполняется:
  - проверить роль пользователя и права;
  - проверить статус ревизии (недопустимые переходы блокируются с `409`).
